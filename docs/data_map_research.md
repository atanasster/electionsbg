**STATUS: BUILT 2026-06-10** (same day, v1 + refinement rounds). Shipped as designed with these deltas:

Prerender + sitemap entries added, `vendor-flow` manual chunk keeps React Flow (41 KB gz) off every other page.
- **The map IS the `/data` landing page** (restructure same day): the old combined DataScreen was split — `/data` = map (with `DataNav` pill row), `/data/sources` = full source registry + downloads (`DataSourcesScreen`), `/data/updates` = refresh log (`DataUpdatesScreen`). Redirects: `/data/map` → `/data` (preserves `?node=&view=`), `/data-changes` → `/data/updates`.
- **Canvas sizes to the graph's aspect ratio** (width-driven, height extends proportionally, capped ~1.15×): initial zoom lands near 1:1 on desktop instead of fit-everything-tiny in a landscape box. ELK edge-routing channels are zeroed out (React Flow draws its own beziers), shrinking the graph to 968×1564. The camera no longer jumps on selection — dimming + animated arrows carry the lineage in a page-scrolled tall figure; it only re-fits on pane resize (useStore width/height subscription — fitView alone runs just once on mount).
- Known gotchas hit: React Flow root collapses to 0 height inside a flex item (fixed with an absolute-fill wrapper); a fresh dep needs `rm -rf node_modules/.vite` or dev serves a stale optimizer cache ("Invalid hook call").
- Not built yet (v2+ candidates): lenses (recolor by cadence/origin), walkthrough tours, AI-tool tier with `reads:` annotations, double-click ego-focus.

## Concept

A public-facing, bilingual interactive diagram — working title **„Картата на данните" / The Data Map** — that shows the full chain:

```
външни източници (51)  →  пайплайни/skills (26)  →  данни (artifacts)  →  функции на сайта (~13 families)  →  AI чат tools (75+)
```

The core message for visitors: *one ingested source powers many features*, and everything on Наясно traces back to a verifiable public origin. The same diagram doubles as living internal documentation.

The inspiration reference (Workspaces-style stack diagram) supplies the aesthetic grammar: dark canvas, dashed tier containers, rounded nodes with a title + muted detail lines, labeled bezier edges, a chip row that switches "views" by dimming everything not in the active view, and a focused node highlighted with an accent outline + a contextual detail card.

## Why this is cheap for us: the graph already exists in machine-readable form

| Layer | Source of truth | Status |
|---|---|---|
| External sources (51) | `scripts/watch/sources/index.ts` (watcher registry) | exists, maintained |
| Source → skill edges | process-watch-report mapping table | exists, maintained |
| Skill → artifact edges | `.claude/skills/update-*/SKILL.md` (prose) | needs a small structured `outputs:` declaration or a curated table |
| Artifact → feature edges | `src/data/*/use*.ts` hooks + `src/routes.tsx` | derivable; v1 = curated feature-family map (~13 families) |
| Feature → AI tool edges | `ai/tools/registry.ts` (75+ tools, no `reads:` field) | v2: add `reads:` annotation or build-time regex over `fetchData()` calls |
| Freshness | `state/watch/<source>.json` `lastChanged` + `state/ingest/<skill>.json` | exists — enables live "updated today" badges |

Extensibility consequence: the diagram is **generated, not drawn**. A new watcher source automatically becomes a new node; CI can assert every source has a node and every node has at least one outgoing edge (no orphans).

## Verified research findings → decisions

### Rendering library: @xyflow/react (React Flow)

- Production-validated for exactly this genre: **Stripe builds its docs diagrams with React Flow** ("diagrams built with React Flow can easily incorporate interactivity and clickable links" — stripe.dev/blog/markdoc). [verified 6-0]
- Custom node = a React component → we style nodes with Tailwind + our navy/coral HSL variables, reuse i18n, link nodes to routes. The screenshot aesthetic is reproducible 1:1 as a custom node type.
- Pan/zoom, touch, minimap, a11y, edge rendering all come free; the lib is modest and the route is lazy-loaded anyway.
- Rejected alternatives:
  - **d3-force / ConnectionsCanvas reuse** — force layout is the wrong genre for tiered lineage (no stable left-to-right reading order).
  - **Mermaid / diagrams-as-code renderers** — auto-layout only, no dim/highlight API, manual positioning open since 2015 (mermaid#270). [verified]
  - **Hand-laid custom SVG** (what the inspiration screenshot likely is) — most beautiful per-pixel, but breaks the "generated, extensible" requirement at 51+ sources. We instead allow optional per-node `pin: {x,y}` overrides in the manifest for hand-tuning.

### Layout: ELK "layered" (Sugiyama), precomputed offline

- elkjs is the only engine in React Flow's own comparison supporting **all three** of: edge routing, group/sub-flow layout, dynamic node sizes. dagre has no edge routing and an open compound-graph bug (dagrejs/dagre#238, open since 2018) that breaks grouped tiers — a direct risk for our dashed tier containers. [verified 12-0 / 6-0]
- elkjs costs ~1.45 MB minified (~433 KB gz) — unacceptable in the client. **Resolution: we are JAMstack with an offline pipeline — run elkjs at build time in `scripts/data_map/` and ship positioned JSON.** Zero client bundle cost, and layout becomes deterministic/reviewable in git diffs.
- Precompute **two layouts**: `elk.direction: RIGHT` (desktop) and `DOWN` (mobile portrait). Both ship in the manifest.
- Bilingual node sizing: measure BG and EN labels at build time (canvas measureText or font metrics), size each node to the max of the two → one layout serves both languages. If divergence ever gets ugly, precompute per-language positions (cheap, it's offline).

### Interaction model: copy three proven patterns

1. **Ilograph "perspectives" → view chips** (the chip row in the screenshot): Всичко / Избори / Парламент / Бюджет / Поръчки и фондове / Местна власт / Цени / Риск / AI. A chip activates a *view* = a tagged subgraph; everything else dims to ~15% opacity. Views are declarative entries in the manifest (model/views separation — the Structurizr/Ilograph pattern, verified 9-0). **Fixed topology + dimming, no re-layout** — at ~70 visible nodes we sit below the scale where dbt/DataHub are forced into subgraph selectors [verified 9-0], and fixed topology preserves the user's mental map between chips.
2. **dbt Explorer → select-to-highlight + detail panel + Lenses**: click a node → highlight its full upstream/downstream closure, open a right-hand (bottom-sheet on mobile) detail card: description, external origin link, update cadence, last-updated (live from `state/watch`), артефакти produced, features powered, "open feature" / "download data" links. **Lenses** = recolor/badge toggles over the same topology: by update cadence (daily/monthly/quarterly/annual), by freshness (updated today/this week/stale), by origin type (state institution / EU / NGO / scraped). [verified 9-0]
3. **DataHub → ego-centric focus mode**: double-click (or tap on mobile) re-centers on one node showing only its closure — the mobile fallback when the full canvas is too dense for 380 px. [verified 9-0]
4. **Walkthrough tours** (Ilograph): a scripted story mode — e.g. „Как един протокол на ЦИК става карта" or „Откъде идват цените" — steps through nodes with narration. Great Facebook-post material. v3.

### Data model: typed JSON manifest, no DSL

Verified pattern (Structurizr DSL model/views split, dbt manifest.json with parent_map/child_map): a DSL adds a parsing step without adding capability. Schema sketch:

```jsonc
// public/data_map.json — generated by scripts/data_map/build_manifest.ts
{
  "version": 1,
  "generated": "2026-06-10",
  "nodes": [
    {
      "id": "src:kzp_prices",
      "type": "source",            // source | pipeline | artifact | feature | aitool
      "label": { "bg": "КЗП „Колко струва"", "en": "KZP price monitor" },
      "detail": { "bg": "1.45М реда/ден, 207 вериги", "en": "1.45M rows/day, 207 chains" },
      "href": "https://kolkostruva.bg",        // external origin or internal route
      "tags": ["prices", "consumption"],
      "cadence": "daily",
      "origin": "state",
      "freshness": "2026-06-10T04:00:00Z"      // injected from state/watch at build
    }
  ],
  "edges": [{ "from": "src:kzp_prices", "to": "art:prices", "label": "daily ZIP" }],
  "views": [
    { "id": "prices", "label": { "bg": "Цени", "en": "Prices" }, "tags": ["prices"] }
  ],
  "layout": { "desktop": { "src:kzp_prices": { "x": 0, "y": 840 } }, "mobile": { } }
}
```

Granularity control: 51 sources is too many boxes for the default view. The manifest supports `group` nodes (dashed containers, like the screenshot's CLIENTS/APP WORKER/BACKENDS) — default view shows ~14 source groups (ЦИК, parliament.bg, Eurostat, НСИ, data.egov.bg, Сметна палата, ИСУН, АОП, ГРАО, КЗП, общини, НОИ/МФ, NGO/TI/ИПИ, Wikipedia); a chip or zoom level expands a group into its constituent sources (progressive disclosure).

## Aesthetic spec (from the inspiration + our brand)

- Dark navy canvas (existing HSL vars from src/index.css), coral accent for the focused node ring + active chip.
- Tier containers: 1px dashed, low-contrast, uppercase letter-spaced tier titles (ИЗТОЧНИЦИ / ПАЙПЛАЙН / ДАННИ / ФУНКЦИИ).
- Nodes: rounded-lg, subtle border, title + 1–3 muted mono detail lines; feature nodes get the coral-tinted border treatment.
- Edges: bezier, low-opacity default, mid-edge labels only on highlight; animated dash on the active lineage path.
- Bottom-right contextual card (like the screenshot's "Cloudflare Artifacts" card) = the dbt-style detail panel.
- Dimming: opacity transition ~200ms; never remove nodes on chip switch (fixed topology).

## Where it lives

- New route `/data/map` (deep-linkable, prerendered for SEO, added to sitemap + internal links from /data and the footer). `/data` #sources tab gets a teaser tile linking to it.
- URL state: `?view=prices&node=src:kzp_prices` — shareable deep links into a focused view (matches our existing URL-contract conventions).

## Phases

1. **Manifest generator** (`scripts/data_map/build_manifest.ts`): read watcher registry + source→skill map + curated skill→artifact + artifact→feature tables; inject freshness; run elkjs layered layout (RIGHT + DOWN); emit `public/data_map.json`. CI orphan check.
2. **Page** (`/data/map`): React Flow with custom node/edge/group components, chip row, click-to-highlight closure, detail panel, BG/EN, dark/light. Lazy route.
3. **Lenses + ego focus + freshness badges**: cadence/freshness/origin recoloring, double-click re-center, mobile bottom-sheet, live "обновено днес" pulse.
4. **Tours + AI-tool tier**: walkthrough story mode; add `reads:` to ai/tools registry (or regex pass) and an AI chip; naiasno-post FEATURE announcement.

## Open questions (from verification)

- Chip switch: dim-over-fixed-topology (recommended) vs re-layout of visible subgraph — no source compared the two UX-wise; fixed topology chosen for mental-map stability and zero client layout cost.
- React Flow touch behavior at 380px (pan/zoom vs page scroll, tap-as-hover) — needs a quick prototype; ego-focus mode is the designed fallback.
- tldraw / isoflow / D2 produced no verifiable claims in the research pass — evidence gap, not a verdict; none threaten the React Flow + offline ELK recommendation.
